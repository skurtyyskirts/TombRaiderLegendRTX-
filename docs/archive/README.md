# Archive

Historical artifacts from early development sessions. Not actively maintained — preserved for reference only.

For current project state, see [`docs/status/WHITEBOARD.md`](../status/WHITEBOARD.md).

---

## Contents

### `sessions/`

Session handoff documents and exploratory analysis from the earliest development phases (pre-build-016). These predate the current `CHANGELOG.md` + `docs/status/` workflow. They document the first attempts at FFP proxy design, early engine analysis, and the initial culling investigation.

Notable files:
- `TOMB_RAIDER_LEGEND_RTX_REMIX_HANDOFF.md` — first formal session handoff document
- `VIBREVERSE-*.md` — early reverse engineering session outputs (paper, roadmap, deep analysis)
- `FIRSTVIBECODE-*.md` — initial exploration: workspace analysis, proxy.ini, rules

### `prompts/`

AI prompt engineering files from early sessions. Used to bootstrap Claude Code / Cursor sessions with project context before CLAUDE.md existed.

- `SESSION_PROMPTS.md` — ready-to-paste Claude Code session starters (superseded by CLAUDE.md)
- `dx9-ffp-port.prompt.md` — early DX9 FFP port skill prompt (superseded by `.claude/rules/dx9-ffp-port.md`)

### `compass/`

Analysis outputs from the Compass AI assistant (early 2026). Used for high-level architecture exploration before the retools static analysis pipeline was established.

### `DEEP_RESEARCH_QUERIES.md`

Research queries sent to Claude.ai deep research mode during the early investigation phase.

### `Combined_Research_Docs.md`

Consolidated dump of early research notes. Superseded by the structured `docs/research/` documents.

## Note on Superseded Documents

Several archived documents have live counterparts in `docs/research/` and `docs/reference/` that are actively maintained. When the archived version and the live version conflict, the live version is authoritative.
